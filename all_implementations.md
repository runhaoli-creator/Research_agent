## Idea #1: DynaCLIP

Build a project called DynaCLIP that learns physics-grounded visual representations for robotic manipulation via contrastive learning where the similarity metric is physical dynamics similarity, not visual or semantic similarity. This idea has been verified novel against 25+ representation learning papers including MCR (ICLR 2025), CLASS (CoRL 2025), PSE (ICLR 2021), DynaMo, R3M, VIP, AFRO, and CLOUD — none of which use dynamics similarity as the contrastive signal. The core insight is that current robotics backbones like DINOv2, CLIP, and SigLIP encode what objects look like, not how they behave physically, and Physics-IQ (DeepMind, arXiv 2501.09038) proves that video models score only 24.1% on physical understanding.

For data generation, use ManiSkill3 (install via pip install mani-skill) which is GPU-parallelized and supports programmatic physics property variation through the SAPIEN API (actor.set_mass(), material.set_static_friction(), material.set_restitution()). Create 50 distinct object geometries from YCB and ShapeNet with 5 texture variants each, giving 250 visual variants. For each geometry, sample 100 random physical property configurations: mass log-uniform in [0.05, 10.0] kg, static friction uniform in [0.05, 1.5], restitution uniform in [0.0, 0.95], dynamic friction = 0.8 times static. This gives 5,000 object-property configurations total. For each configuration, execute 5 standardized diagnostic actions using a Franka Panda arm — push along +X at 0.05 m/s for 1 second, push along +Y, grasp then lift 20cm then release, quick lateral flick at object edge, and slow press downward — recording the resulting object trajectories (3D position, quaternion orientation, 3D linear velocity, 3D angular velocity = 13 dimensions) for 50 timesteps at 20Hz. The dynamics fingerprint for each configuration is the concatenation of all 5 trajectory recordings. Compute pairwise dynamics similarity as the negative average DTW distance (use tslearn.metrics.dtw) across the 5 test actions, normalized to [0,1]. Render 20 RGB images at 224x224 per configuration from varied viewpoints (azimuth 0-360, elevation 15-45 degrees, distance 0.4-0.8m) in 5 different object poses, giving 100,000 total images. Pre-compute DINOv2 embeddings for all images and mine pairs: 30% hard negatives (DINOv2 cosine similarity above 0.9 but dynamics similarity below 0.3), 30% hard positives (DINOv2 cosine similarity below 0.5 but dynamics similarity above 0.7), and 40% random. Also create an Invisible Physics test set of 500 image pairs where the objects use the same mesh, same texture, and same viewpoint (so they are visually identical with DINOv2 cosine similarity above 0.99) but have different physical properties.

The DynaCLIP model uses DINOv2-ViT-B/14 (86M parameters, loaded via torch.hub.load facebookresearch/dinov2 dinov2_vitb14) as the backbone, extracting the CLS token concatenated with mean-pooled patch tokens to get a 1536-dimensional vector, followed by a projection head of Linear(1536, 768) then LayerNorm then GELU then Linear(768, 512) with L2 normalization, producing a 512-dimensional unit-norm embedding. The backbone is UNFROZEN during pre-training to reshape the feature space from semantic to dynamics similarity. The projection head is discarded after pre-training. Train with a Soft InfoNCE loss: given embeddings z_i and z_j for a batch of B pairs and the B-by-B pairwise dynamics similarity matrix, compute logits as z_i times z_j transpose divided by temperature, compute soft labels as softmax of the dynamics similarity matrix divided by 0.1, and minimize the cross-entropy between soft labels and log-softmax of logits. Use AdamW with backbone learning rate 1e-5 and projection head learning rate 1e-3, weight decay 0.05, cosine annealing schedule with 500 warmup steps, batch size 1024 pairs across 8 GPUs, learnable temperature initialized at 0.07, train for 100K steps in bf16 precision, with augmentation of RandomResizedCrop at scale 0.8-1.0, RandomHorizontalFlip, and ColorJitter at 0.1.

Datasets for evaluation: (1) LIBERO-10 with 50 demos per task, 10 tasks, reporting average success rate over 100 episodes times 3 seeds, (2) LIBERO-Long for long-horizon manipulation, (3) CALVIN ABC-D with 34 tasks and 5-step instruction chains reporting average completed chain length, (4) ManiSkill3 with physics variation across Push, PickAndPlace, Stack tasks, (5) a custom Physics-Varying Benchmark where training uses mass 0.5-2.0 kg and friction 0.5-1.0 but testing uses mass 0.1 and 5.0 kg and friction 0.1 and 1.5, and (6) RLBench-18 with 18 tasks and 100 demos per task variation.

Backbones compared as frozen visual encoders (8 total): (1) DynaCLIP (ours, DINOv2-ViT-B/14 fine-tuned with dynamics contrastive), (2) DINOv2-ViT-B/14 pretrained frozen, (3) DINOv2-ViT-L/14 pretrained frozen, (4) SigLIP-ViT-B/16 from google/siglip-base-patch16-224, (5) CLIP-ViT-L/14 from openai/clip-vit-large-patch14, (6) R3M loaded via pip install r3m then r3m.load_r3m resnet50, (7) VIP loaded via pip install vip, and (8) MCR pretrained on DROID from arXiv 2410.22325.

Baselines for downstream policy evaluation (7 total): train a Diffusion Policy from Cheng Chi et al. 2023 with 16-step action chunks and 7-DoF actions using DDPM 100 steps for training and DDIM 10 steps for inference as the primary policy, and compare DynaCLIP-backed policy against: (1) DINOv2-backed Diffusion Policy, (2) R3M-backed Diffusion Policy, (3) OpenVLA-OFT from arXiv 2502.19645 as a state-of-the-art VLA baseline, (4) Octo-Base from arXiv 2405.12213 as a generalist policy baseline, (5) ACT (Action Chunking with Transformers) from Zhao et al. 2023, (6) Dreamer-v3 from danijar/dreamerv3 as a model-based RL baseline, and (7) TD-MPC2 from nicklashansen/tdmpc2.

Run 5 experiments. Experiment 1 Physics Property Linear Probing: train a single linear layer on frozen representations from all 8 backbones to predict mass (MSE loss, report R-squared), friction (same), restitution (same), and material category (10 classes, report accuracy) on held-out data, with 5 seeds and 95% confidence intervals, expected DynaCLIP R-squared above 0.8 while others below 0.3. Experiment 2 Invisible Physics Test: on the 500 visually identical pairs, report the distribution of cosine similarities under each encoder, binary classification accuracy for which object is heavier, and train a Diffusion Policy on a grasp-and-lift task with varying object mass and show DynaCLIP-backed policy adjusts grasp force while DINOv2-backed fails. Experiment 3 Downstream World Model: train a Dreamer-v3 style RSSM with 512-dim stochastic state using 32 categoricals times 32 classes and 512-dim deterministic GRU state using each of the 8 backbones as frozen visual encoder on ManiSkill3 push, pick-and-place, and stack with physics variation at 10K trajectories each, and report latent MSE at horizons t+1, t+5, t+10, t+20, reconstruction SSIM and LPIPS, FVD over 16-frame clips, object position L2 error, and physics violation rate. Experiment 4 Downstream Policy on all 6 datasets: train Diffusion Policy with each backbone on LIBERO-10, LIBERO-Long, CALVIN ABC-D, ManiSkill3, Physics-Varying Benchmark, and RLBench-18, and compare against all 7 policy baselines. Experiment 5 Zero-Shot Physics Inference: build a library of 1000 objects with known properties, for each query image find 5 nearest neighbors in each backbone's embedding space, predict properties as similarity-weighted average, report R-squared.

Run 8 ablation studies: (1) dynamics similarity metric comparing DTW versus endpoint L2 versus full trajectory MSE versus velocity-only DTW, (2) number of diagnostic test actions K in 1, 3, 5, 10, (3) contrastive loss formulation comparing Soft InfoNCE versus standard binary InfoNCE versus triplet loss versus BYOL-style non-contrastive, (4) hard negative ratio in 0%, 15%, 30%, 50%, (5) backbone initialization comparing DINOv2 versus ImageNet versus random, (6) data scale at 10K, 25K, 50K, 100K images plotting a scaling curve, (7) physical property diversity comparing mass only versus friction only versus mass plus friction versus all properties, and (8) fine-tuning depth comparing freeze all versus unfreeze last 2 layers versus last 4 versus all. Finally, produce analysis: t-SNE and UMAP visualizations of DynaCLIP versus DINOv2 embedding spaces colored by object category versus mass versus friction, Jacobian analysis of partial z over partial mass and partial z over partial friction showing DynaCLIP is sensitive to physics changes while DINOv2 is near zero, cross-simulator transfer from ManiSkill3 pretraining to Isaac Lab evaluation, and computational cost analysis showing DynaCLIP adds zero inference overhead since it is the same architecture as DINOv2 with different weights.

## Idea #2: PhysBridge

Build a project called PhysBridge that adapts a pretrained physics foundation model to robot manipulation and shows that physics-first pretraining outperforms video-first pretraining for manipulation world models. This idea has been verified novel — PhysiX (4.5B params, arXiv 2506.17774), Walrus (1.3B, arXiv 2511.15684), and GPhyT (1.8TB data, arXiv 2509.13805) all exist as physics foundation models but none has been adapted to robot manipulation. The core claim is that adapting a physics FM to manipulation yields better data efficiency and physical reasoning than adapting a video FM like DreamZero or Cosmos, because physics understanding (forces, contacts, Newton's laws) transfers directly while video pattern-matching does not.

The architecture has three modules. Module 1 is a Visual Perception Adapter that converts RGB 224x224 images into physics-interpretable state tokens: use a frozen visual backbone producing patch tokens, then a 6-layer Transformer decoder with 10 learned object queries that cross-attend to the patch tokens and output per-object state vectors containing 3D position, 3D velocity, 32-dimensional shape embedding, 4D quaternion orientation, and 4D bounding box totaling 46 dimensions per object, projected to the physics FM token dimension, trained with supervision from simulator ground-truth object states. Module 2 is the Physics Foundation Model Backbone: either download pretrained GPhyT weights from arXiv 2509.13805 (hybrid neural-differentiator plus numerical-integrator) or PhysiX weights from arXiv 2506.17774 (4.5B autoregressive physics token predictor), or if neither is available, train a fallback GNN-based dynamics model following the Graph Network Simulator architecture from Sanchez-Gonzalez et al. 2020 with 8-layer GNN using Transformer-style message passing, 512-dimensional node features, 128-dimensional edge features, approximately 200M parameters, trained on 10M+ diverse non-robot physics episodes from Isaac Lab covering rigid body dynamics (2M episodes), multi-body interactions (500K episodes), and contact dynamics (500K episodes), with objects of mass 0.01 to 50 kg and friction 0.01 to 2.0. Insert LoRA adapters at rank 16 into each attention layer of the physics FM, keeping backbone weights frozen during manipulation fine-tuning. Module 3 is a Robot Action Adapter: a 2-layer MLP mapping 7-dimensional actions through 256 hidden units with SiLU activation to the physics FM token dimension, injected as an intervention node.

Visual backbones for the perception adapter (4 options, ablated): (1) DINOv2-ViT-B/14 frozen, (2) DINOv2-ViT-L/14 frozen, (3) SigLIP-ViT-B/16 frozen, and (4) DynaCLIP-ViT-B/14 if available from Idea 1.

Datasets for evaluation (5 total): (1) LIBERO-10 with 50 demos per task reporting success rate over 100 episodes times 3 seeds, (2) LIBERO-Long for long-horizon tasks, (3) CALVIN ABC-D with 24 hours play data plus 20K annotated trajectories reporting average 5-step chain length, (4) ManiSkill3 with Push, PickAndPlace, Stack, PegInsert at 5K trajectories per task with physics variation, and (5) RLBench-18 with 100 demos per task variation.

Baselines (8 total): (1) Cosmos Policy from NVIDIA arXiv 2601.16163 representing the video FM to manipulation path which is the most critical comparison, (2) DreamZero-style World Action Model by fine-tuning Cosmos-Predict2.5-2B or Stable Video Diffusion jointly on video plus action prediction, (3) Dreamer-v3 from official danijar/dreamerv3 trained from scratch, (4) TD-MPC2 from nicklashansen/tdmpc2, (5) SimDist from arXiv 2603.15759 which pretrains in robot-specific simulation, (6) OpenVLA-OFT from arXiv 2502.19645 as a state-of-the-art VLA without world model, (7) PhysBridge-NoPretraining with the same architecture but physics backbone initialized randomly to isolate pretraining contribution, and (8) PhysBridge-VideoInit with the physics backbone initialized from video FM weights.

Run 6 experiments all in simulation. Experiment 1 Data Efficiency which is the key result: fix LIBERO-10 as target, vary fine-tuning data at 5%, 10%, 25%, 50%, 100% of available demos, plot success rate versus data fraction for all methods, expected result PhysBridge at 10% matches DreamZero at 100%. Experiment 2 Multi-Benchmark Evaluation with full data on all 5 datasets, 100 episodes times 3 seeds per task. Experiment 3 Physics Generalization: train on standard physics then test on extreme physics with mass 10 times heavier and friction 5 times lower. Experiment 4 World Model Prediction Quality: object position L2 error at horizons t+1, t+5, t+10, t+20, t+50, physics violation rate, contact prediction accuracy. Experiment 5 Cross-Simulator Transfer: train on ManiSkill3, evaluate on LIBERO. Experiment 6 Downstream Policy Comparison: use PhysBridge world model for MPC planning with Diffusion Policy, compare against OpenVLA-OFT, Octo-Base, ACT, and Dreamer-v3 actor-critic.

Run 8 ablations: (1) physics FM backbone comparing GPhyT versus PhysiX versus our GNN versus random, (2) adapter type comparing LoRA versus full fine-tune versus frozen, (3) perception adapter backbone comparing DINOv2-B versus DINOv2-L versus SigLIP-B versus ground-truth oracle, (4) physics pretraining data diversity rigid-only versus rigid plus multi-body versus all, (5) pretraining data scale at 100K, 500K, 1M, 3M episodes, (6) action injection comparing intervention node versus concatenation versus cross-attention, (7) LoRA rank controlling trainable params at 1M, 5M, 20M, 50M, and (8) perception error tolerance. Analysis: probe features before and after fine-tuning, CKA similarity between physics FM and video FM representations, failure analysis on deformable objects, and scaling from 200M to 1B physics FM.

## Idea #3: PhysSteering

Build a project called PhysSteering that discovers and steers emergent physics features in robot world models using Sparse Autoencoders. The Walrus Physics Steering paper (arXiv 2511.20798) showed that physics foundation models learn abstract transferable physical concepts that can be extracted and steered. The Stanford SAE paper (arXiv 2603.19183) applied Sparse Autoencoders to VLAs and found steerable motion primitives. Nobody has combined these to discover and steer physics features in robot world models like DreamZero, Dreamer-4, or Cosmos. Verified novel.

Generate physics-varying manipulation data in ManiSkill3: 30 object types times 100 physical property configs across mass, friction, and restitution, across 6 tasks (Push, PickAndPlace, Stack, Slide, LiftAndDrop, PegInsert) with 50 trajectories per task per config giving 900K total trajectories, recording RGB frames, actions, proprioception, object states, and ground-truth physics parameters. Train reproduction world models at 300-500M parameters: (1) a DreamZero-style DiT-based world action model, (2) a Dreamer-v3-style RSSM model, (3) a Cosmos-style flow-matching model, and (4) a UWM-style unified diffusion model from arXiv 2504.02792, all trained on the ManiSkill3 data WITH physics variation. If the open-sourced DreamZero-14B is computationally feasible, also use it as the primary target. Run forward passes on all trajectories, extract hidden activations at every layer and every timestep, store as memory-mapped numpy arrays.

Train Sparse Autoencoders per layer: input activation h of dimension d_model, encode through Linear(d_model, M) with bias followed by ReLU producing sparse features f, decode through Linear(M, d_model) with bias, train with MSE reconstruction plus lambda times L1 sparsity on f. Sweep dictionary size M in 1K, 4K, 16K, 64K and lambda in 1e-4, 3e-4, 1e-3. Adam lr 3e-4, 100K steps, batch 4096. Approximately 112 SAEs per model. For each feature compute Pearson correlation with ground-truth mass, friction, restitution, velocity, and contact force, flagging features with absolute r above 0.5 and p below 0.001. Test steerability by clamping a discovered mass feature to a different activation level and measuring whether predicted dynamics change consistently.

Datasets for evaluation (4 total): (1) ManiSkill3 with physics variation as primary, (2) LIBERO-10 with 50 demos per task, (3) CALVIN ABC-D with 5-step chains, and (4) RLBench-18 with 100 demos per variation.

Backbones / target world models (4 total): (1) DreamZero-style DiT (300M-14B), (2) Dreamer-v3-style RSSM (300M), (3) Cosmos-style flow-matching (300M-2B), and (4) UWM-style unified diffusion (300M).

Baselines (8 total): (1) unsteered world model no adaptation, (2) fine-tuned with 100 gradient steps, (3) LoRA-adapted with 50 steps, (4) random feature steering as control, (5) AdaptiGraph gradient-based physics optimization from arXiv 2407.07889, (6) DALI context-conditioned adaptation from arXiv 2508.20294, (7) OpenVLA-OFT as direct policy without world model from arXiv 2502.19645, and (8) Diffusion Policy with DINOv2 backbone as non-adaptive policy baseline.

Run 6 experiments. Experiment 1 Physics Feature Discovery: produce per-layer heatmaps of feature-by-physics-property correlation for each world model architecture, report number of physics features per layer, selectivity, and cross-task consistency. Experiment 2 Steerability Quantification: for each discovered physics feature, sweep steering strength at 0.5x, 1x, 2x, 5x of mean activation, plot dynamics change magnitude, confirm monotonicity. Experiment 3 Adaptation Speed: on ManiSkill3 with novel physics (5x heavier, 5x lower friction), compare success rate and wall-clock adaptation time for steering at 10ms versus fine-tuning at 5s versus LoRA at 3s versus AdaptiGraph versus DALI, evaluated on 100 episodes times 3 seeds. Experiment 4 Cross-Task Transfer: discover mass and friction features from push task, test steering on pick-and-place, stack, and peg-insert, report per-task success rate on LIBERO-10 and CALVIN. Experiment 5 Cross-Architecture Comparison: compare physics feature emergence across DreamZero-DiT, Dreamer-RSSM, Cosmos-flow, UWM-diffusion, report which develops cleanest and most steerable features. Experiment 6 Downstream Policy: use steered world models for MPC planning and compare final manipulation success on LIBERO-10, CALVIN, RLBench-18 against all baselines.

Run 8 ablations: (1) dictionary size 1K, 4K, 16K, 64K, (2) sparsity lambda, (3) physics-varied versus fixed-physics training data, (4) model scale 300M, 1B, 2B, 14B for minimum scale producing physics features, (5) layer depth analysis early versus mid versus late, (6) SAE features versus PCA directions versus random directions, (7) training object diversity at 5, 15, 30, 50 objects, and (8) steering method comparing clamping versus addition versus linear vector. Analysis: t-SNE colored by physics properties, feature knockout study, qualitative predicted videos with and without steering, and compute cost comparison.

## Idea #4: PhysContext

Build a project called PhysContext that implements in-context physical property learning for manipulation world models, where the model learns an object's physics from watching a single diagnostic interaction with zero fine-tuning and zero gradient updates, purely through in-context learning in the Transformer's context window. Note that DALI (NeurIPS 2025, arXiv 2508.20294) has 75% overlap — it does context-conditioned world model adaptation without gradients on MetaWorld using a latent context vector. The critical differentiator is compositional transfer: given object A with mass 2kg and friction 0.3 and object B with mass 0.5kg and friction 1.0, PhysContext composes physics to predict novel object C with mass 2kg from A and friction 1.0 from B that was never seen during training. DALI's opaque latent context cannot do this. This compositional transfer experiment is the make-or-break experiment.

The architecture is a causal Transformer with 12 layers, hidden dimension 512, 8 attention heads, FFN dimension 2048, SiLU activations, RMSNorm pre-normalization, Rotary Position Embeddings, max context length 4096 tokens, approximately 85M parameters. The input sequence: CTX_START token, K diagnostic frames each of 256 visual tokens from frozen DINOv2-ViT-B/14 projected to 512-d plus 1 action token, CTX_END token, then 256 current observation tokens, 1 proprioception token, 1 action token, predicting next 256 visual tokens. Segment embeddings distinguish context (segment 0) from current (segment 1). Loss: L2 in DINOv2 feature space plus 0.05 times pixel reconstruction. Train with AdamW lr 3e-4, cosine schedule, 2000 warmup, 500K steps, batch 128, bf16, context curriculum K=10 then K=5 then K=uniform(1,10).

Datasets (5 total): (1) ManiSkill3 with 30 objects times 100 property configs, 5 diagnostic actions, 6 tasks, 20 trajectories per config, split by property combination 70/15/15, (2) LIBERO-10 with 50 demos per task, (3) LIBERO-Long, (4) CALVIN ABC-D with 5-step chains, and (5) RLBench-18 with 100 demos per variation.

Visual backbone options (4, ablated): (1) DINOv2-ViT-B/14 as primary, (2) DINOv2-ViT-L/14, (3) SigLIP-ViT-B/16, and (4) CLIP-ViT-L/14.

Baselines (9 total): (1) No-Context same Transformer with zero vector replacing context, (2) DALI reproduced from arXiv 2508.20294 which is the critical baseline, (3) AdaptiGraph gradient-based optimization at 10, 50, 100 steps from arXiv 2407.07889, (4) Oracle-Numerical with ground-truth physics parameters as direct input, (5) Dreamer-v3 from danijar/dreamerv3, (6) TD-MPC2 from nicklashansen/tdmpc2, (7) Average-Context with same average-physics context always, (8) Random-Context with context from a random different object, and (9) OpenVLA-OFT from arXiv 2502.19645 as direct policy baseline. For downstream policy, use Diffusion Policy with MPC planning through PhysContext world model, and also compare against ACT and Octo-Base.

Run 7 experiments. Experiment 1 Compositional Transfer the core differentiator: compose physics from object A and B contexts and predict novel object C, compare PhysContext versus DALI, show DALI fails. Experiment 2 Prediction Quality on novel physics: latent MSE, SSIM, LPIPS, FVD at t+1, t+5, t+10, t+20 on held-out property combinations, compare against DALI, No-Context, Oracle. Experiment 3 Context Scaling: vary K from 0 to 10 and plot prediction error, compare against AdaptiGraph at 0 to 500 gradient steps, show PhysContext K=1 matches AdaptiGraph N=100 and is 500x faster. Experiment 4 Most Informative Diagnostic Action: test each of the 5 actions individually at K=1, report which reveals most about mass, friction, restitution. Experiment 5 Downstream Policy on all 5 datasets: Diffusion Policy with MPC using PhysContext versus all 9 baselines plus ACT and Octo, 100 episodes times 3 seeds on LIBERO-10, LIBERO-Long, CALVIN, ManiSkill3 physics-varying, RLBench-18. Experiment 6 Physics Inference: linear probe on CTX_END hidden state predicting mass, friction, restitution, compare R-squared against DALI's latent context and DINOv2. Experiment 7 Invisible Physics Test: 500 visually identical pairs, KL divergence between predicted dynamics.

Run 8 ablations: (1) context length K from 0 to 20, (2) diagnostic action type, (3) visual backbone DINOv2-B versus DINOv2-L versus SigLIP versus CLIP, (4) model scale 4, 8, 12, 16, 24 layers, (5) context curriculum, (6) training physics diversity at 10%, 25%, 50%, 100% of property combinations, (7) with versus without segment embeddings, and (8) cross-task context using push diagnostics for pick-and-place evaluation. Analysis: attention map visualization from prediction tokens to context tokens, t-SNE of CTX_END hidden states colored by mass and friction, failure case categorization, and latency comparison at 10ms for in-context versus 5s for AdaptiGraph.

## Idea #5: Zero-Success Learning

Build a project called Zero-Success Learning that demonstrates robot manipulation can be learned from ZERO successful demonstrations using only failure data combined with world model imagination. Verified novel — only Grollman and Billard at ICRA 2011 attempted failure-only learning in low dimensions. No modern work combines failure-only data with world model imagination. The core hypothesis is that failed trajectories contain all dynamics information (objects move, contacts happen, friction applies) and the only difference from success is a few critical actions near task completion.

The method is a three-stage pipeline. Stage 1 trains a world model on failure-only data using a Dreamer-v3 style RSSM with a frozen visual encoder projected to 512 dimensions, GRU with 512-dim deterministic state, stochastic state with 32 categoricals times 32 classes, transition and posterior and prior MLPs, ConvTranspose decoder, and a goal-proximity reward predictor trained using LPIPS distance to goal image plus object-to-target distance. Stage 2 identifies near-success states where goal proximity exceeds threshold tau. Stage 3 performs imagination-based completion: sample N=256 candidate action sequences of horizon H=10 steps, simulate through world model, select best by goal proximity, also implement CEM with 5 iterations top 10% and MPPI with temperature 1.0. Stitch synthetic successes keeping only those with final proximity above 0.9. Train a Diffusion Policy on synthetic successes.

Visual backbone options for the world model encoder (4, ablated): (1) DINOv2-ViT-B/14 frozen as primary, (2) DINOv2-ViT-L/14 frozen, (3) SigLIP-ViT-B/16 frozen, and (4) R3M ResNet50 frozen.

Datasets (4 total): (1) ManiSkill3 with 6 tasks (Push-to-Target, Pick-and-Place, Stack, Peg-Insert, Drawer-Open, Button-Press) collecting 15K failure trajectories per task using mixed strategies of random policy, scripted exploration, and noisy oracle at sigma 0.3, (2) LIBERO-10 with random policy failures, (3) CALVIN with random play data as failures, and (4) RLBench-18 with random policy failures for 5 selected tasks.

Baselines (9 total): (1) behavioral cloning with N successful demonstrations at N equals 1, 5, 10, 25, 50, 100 per task which is the critical comparison, (2) BC with 0 demos as random policy lower bound, (3) Dreamer-v3 from danijar/dreamerv3 with online RL from scratch, (4) Hindsight Experience Replay with goal relabeling, (5) Plan2Explore then fine-tune via world model disagreement, (6) random augmentation of failure trajectories without world model imagination, (7) DAgger with simulator oracle corrections, (8) OpenVLA-OFT from arXiv 2502.19645 fine-tuned with N demos as VLA baseline, and (9) Zero-Success without near-success filtering to test whether near-success identification matters. For downstream policy comparison, also report against ACT and Octo-Base at matching demo budgets.

Run 7 experiments. Experiment 1 N-Success Learning Curve which is the key result: plot success rate versus number of successful demonstrations from 0 to 100 for Zero-Success (ours at N=0), BC, Dreamer-v3, HER, DAgger, OpenVLA-OFT, ACT, Octo on all 4 datasets with 100 episodes times 3 seeds, expected Zero-Success at N=0 achieves 75% matching BC at N=25. Experiment 2 Failure Data Composition: compare random versus scripted versus noisy-oracle versus mixed, report which produces most near-success states and best final policy. Experiment 3 Near-Success Statistics: fraction of failures with near-success states at thresholds 0.5 to 0.9, distance in timesteps from actual success, visualization. Experiment 4 World Model Quality: compare prediction accuracy of world model trained on failures-only versus successes-only versus mixed on same held-out test set, expected dynamics prediction quality is comparable because dynamics are the same. Experiment 5 Imagination Quality: visualize imagined completions, measure physical plausibility via penetration and gravity violation rate, goal achievement rate, diversity of solutions. Experiment 6 Failure Data Scaling: vary failure data at 1K, 5K, 15K, 50K trajectories per task and plot success rate. Experiment 7 Multi-Benchmark Final Comparison: full results table on ManiSkill3 all 6 tasks, LIBERO-10, CALVIN, RLBench-18 selected tasks, comparing Zero-Success against all 9 baselines.

Run 8 ablations: (1) near-success threshold tau from 0.5 to 0.9, (2) imagination horizon H at 3, 5, 10, 20 steps, (3) search method random N=256, 1024, 4096 versus CEM versus MPPI, (4) completions per near-success at 1, 5, 10, 50, (5) world model architecture RSSM versus Transformer versus diffusion-based, (6) goal proximity function LPIPS versus SSIM versus learned MLP versus object distance, (7) failure data quality random versus scripted versus noisy-oracle versus mixed, and (8) iterative refinement 1 versus 2 versus 3 rounds. Analysis: what failures teach visualization showing world model's learned dynamics, critical decision point analysis showing where near-success states occur, failure taxonomy categorizing utility of different failure types, information-theoretic analysis of mutual information between failure data and task success, and failure scaling law plot. Risk mitigation: if N=0 produces below 50% success, pivot to Few-Success showing 5 demos plus failure data matches 50 demos BC, contribution becomes failures multiply demo value 10x.
